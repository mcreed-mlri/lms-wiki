Phase 0: In-Course HTML Wrappers Largely Complete

Clean View breakthrough: append ?ou={orgUnitId}&d2l_body_type=3 to Brightspace topic URLs. This is the gold-standard wrapper pattern: Brightspace keeps hosting, access, and tracking, while standard navigation chrome drops away so LACE navigation and styling can own the viewport.
Course Builder cleanup rule: Manage Files stores the actual files; Course Builder/Content stores the outline nodes that point to them. If a file is deleted from Manage Files, remove or replace the matching Course Builder node too, or Valence API outline responses may keep returning broken topics that the LACE Hub discovery layer cannot safely load.

Works Today

  1. Learner opens Brightspace Course Offering.
  2. Learner clicks the LACE launch link.
  3. Custom HTML wrapper opens in clean Brightspace view using d2l_body_type=3.
  4. Learner moves through LACE navigation.
  5. Brightspace records official topic visits/completions, which can flow into Jurisdictional CLE reporting.

Not Phase 0: OAuth, API-backed checkmarks, cross-course dashboard data, manager reporting, or custom catalog sync. Those start in Phase 1 and Phase 2.

Live Spike Routes

  • GET /api/health/supabase/ returns the test learning_items row.
  • GET /api/health/brightspace/ reports Brightspace OAuth configuration status.
  • GET /api/auth/brightspace/start begins the OAuth consent flow.
  • GET /api/health/brightspace/whoami confirms the authorized Brightspace user.
  • GET /api/health/brightspace/course?orgUnitId=6703 validates Eviction Defense course metadata.
  • GET /api/me/dashboard should return learner identity, role permissions, enrollment status, completion %, and Brightspace resume links.

Service User Production Path

  1. Create the non-human Brightspace Service User. Do not bind production sync to a personal admin account.
  2. Create a least-privilege role such as Hub Sync Service with read-only metadata permissions.
  3. Enroll the Service User in the Org Units it needs for catalog sync.
  4. Register the production OAuth app in Manage Extensibility using Client Credentials if available.
  5. Associate the app with the Service User and publish an HTTPS JWKS URL for JWT assertion verification.
  6. Implement the Next.js token request with short-lived JWT client assertions and grant_type=client_credentials.

Reusable Takeaways

  • Competency-grounded outcomes: define what good performance looks like, not only what topics are covered.
  • Practice evidence: connect skills and assessments to real-world legal practice.
  • Outcome-to-assessment chain: goals, terms, objectives, rubrics, and completion criteria should line up.
  • Incremental improvement: map the current curriculum, find gaps, and improve without implying a full reboot.
