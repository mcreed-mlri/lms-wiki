Brightspace Native Fallback Make Presentable

Design stance: LACE Learning Hub remains the intended front door. Brightspace still needs to look intentional because learners may enter through Brightspace login, bookmarks, Pulse, emails, notifications, or admin links.

Treat the native Brightspace homepage as a fallback lobby, not the primary learning product. It does not need to be highly custom, but it cannot look empty, buried, or under construction.

Minimum Standard

  • Move My Courses, Continue, or Start Here as high as Brightspace allows.
  • Remove, hide, or collapse unused widgets such as placeholder announcements, social blocks, app promos, empty auditor widgets, and admin-only clutter.
  • Add calm orientation copy: Welcome to MLRI Learning. Use this page to access your assigned courses.
  • Add a clear path back to LACE Learning Hub for search, recommendations, resume, and dashboards.
  • Keep navbar links minimal: My Courses, LACE Hub, Help, and profile/account controls.
  • Make each native course homepage consistent: banner, Start/Resume button, short course description, support link.
  • Check mobile so the first task is reachable without scrolling through irrelevant widgets.

Brightspace Manager In Progress

Product role: Brightspace Manager is the command center for the platform build. It handles administrative operations, course building, sync checks, file management, and publishing workflows so the learner-facing LACE Learning Hub can stay simpler.

Think of Brightspace Manager as the staff/admin workbench behind the Learning Hub. The Learning Hub remains the front door for learners, managers, and program views; Brightspace Manager is where LACE builds, audits, and maintains the courses and integration data that power that experience.

Live Spike Routes

  • GET /api/health/supabase/ returns the test learning_items row.
  • GET /api/health/brightspace/ reports Brightspace OAuth configuration status.
  • GET /api/auth/brightspace/start begins the OAuth consent flow.
  • GET /api/health/brightspace/whoami confirms the authorized Brightspace user.
  • GET /api/health/brightspace/course?orgUnitId=6703 validates Eviction Defense course metadata.
  • GET /api/me/dashboard should return learner identity, role permissions, enrollment status, completion %, and Brightspace resume links.

Service User Production Path

  1. Create the non-human Brightspace Service User. Do not bind production sync to a personal admin account.
  2. Create a least-privilege role such as Learning Hub Sync Service with read-only metadata permissions.
  3. Enroll the Service User in the Org Units it needs for catalog sync.
  4. Register the production OAuth app in Manage Extensibility using Client Credentials if available.
  5. Associate the app with the Service User and publish an HTTPS JWKS URL for JWT assertion verification.
  6. Implement the Next.js token request with short-lived JWT client assertions and grant_type=client_credentials.

Use Cases: Current Permissions

  • Compliance / completion tracking — query grade values and enrollment data to surface who is current vs. overdue on required trainings, broken down by org unit. Replaces manual admin pulls.
  • Personalized Learning Hub home — pull a learner's active enrollments and in-progress grade data when they open LACE to power a "pick up where you left off" view with live Brightspace data.
  • Cohort-based recommendations — use enrollment:orgunit and organizations:organization to surface what peers in the same role or department have completed.
  • Data Hub export pipeline — bulk exports via datahub:dataexports into a lightweight data store for dashboards and reports without hitting the live API on every query.
  • Grade-gated content unlocking — check whether a learner passed a prerequisite before surfacing advanced content in the Learning Hub.

Use Cases: With Write Permissions

  • Automated enrollment management — highest-value near-term target. New hire onboarding auto-enrolls staff into required courses by role or org unit; no admin touch needed. Role changes and departures trigger enrollment updates automatically.
  • Grade writeback — Learning Hub assessments and scenario exercises post scores back to the Brightspace gradebook so everything lives in one place.
  • Badge and award issuance — issue credentials or completion badges automatically when a learner finishes a Learning Hub pathway, visible in their Brightspace profile.
  • Content creation pipelines — push policy updates or templated course content across multiple courses at once; useful for SOPs and compliance content updates.
  • Targeted announcements — combine enrollment data with announcement write access to send role-specific communications (e.g. nursing staff who haven't completed a required course by a deadline).
  • Competency and outcome tracking — map Learning Hub activities to Brightspace learning outcomes for a fuller picture of staff development beyond course completion.

File and Content Management

The content API supports creating and updating course topics and modules, uploading files to course offerings, and managing SCORM/xAPI package imports. The raw "Manage Files" area has limited API coverage; bulk file reorganization stays manual. The practical pattern for pushing updated policy docs or SOPs across multiple courses: store the canonical file in SharePoint or Drive, then use the content API to update the relevant course topics on change.

For consistent course styling, use a course template with the CSS already applied rather than trying to set a custom CSS file per course via the API. Course creation via API can specify a template, which brings the styling along automatically.
