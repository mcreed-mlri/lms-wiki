Brightspace Platform Map

A friendlier, click-to-explore version of how Brightspace is put together. Open a branch to see what lives inside it — nothing here is required reading top to bottom.

How to Read This Map

Brightspace has a lot of moving parts, but they group into five branches. Start collapsed, open only the branch you care about, and each item carries a one-line plain-language explanation. This is for orientation — the Team Wiki remains the source of truth for LACE's actual setup and naming.

One thing to keep straight: "Org" here means the Brightspace course structure — the site tree where courses live — not MLRI's staff org chart.
Vocabulary note: Brightspace's official default org-unit types are Organization, Department, Semester, Course Template, Course Offering, Group, and Section. LACE labels such as Training Area and Program are planning/catalog labels that may be implemented as Departments, custom org-unit types, or Learning Hub-side categories.
› Organizational Hierarchy Where Brightspace places courses in the site tree. Use it for setup, reporting, and enrollment boundaries. 6 levels
  • Org — the top-level site instance (MLRI).
  • Training Area — broad catalog category, e.g. Legal Skills or Substantive Law.
  • Program — a group of related course templates, e.g. Housing Law.
  • Course Template — the version container that holds all runs of a course.
  • Course Offering — the active, learner-facing course; enroll learners here.
  • Section — optional subdivision for cohorts or scheduled runs.
› Content & File Management Manage Files stores the actual files; Content / Course Builder stores the outline nodes that point to them. 5 items
  • Storage Layer — Manage Files, where the raw files actually live.
› Structure Layer the outline that points at stored files
  • Course Builder — maps outline nodes to files.
  • Modules — unit-level organization of the outline.
  • Topics — the individual content items learners open.
  • HTML Templates — responsive web design for custom pages (LACE's default course model).

Retiring a page? Clean up both the stored file and the Content / Course Builder node that references it.

› Users & Permissions Roles control what a person can do; enrollments and classlists connect people to Course Offerings. 7 items
› Identity Management who someone is and which courses they belong to
  • User Attributes — profile metadata (useful for routing and reporting, with a controlled naming plan).
  • Learning Groups — automation rules for grouping users.
  • Classlist — the participants enrolled in a Course Offering.
› Role Types what each kind of user is allowed to do
  • Learner — standard access; takes the training.
  • Instructor — course authoring and content editing.
  • Manager — progress reporting and oversight.
  • Administrator — full org-level access.
› Developer & Integration Layer Integration plumbing — used for LACE Learning Hub sync, external tools, service-user access, and approved API reads. 6 items
› Extensibility how external apps are registered and authorized
  • Manage Extensibility — register apps that talk to Brightspace.
  • OAuth 2.0 — the security framework for that access.
  • Authorization Scopes — the specific permissions an app is granted.
› Integration Tools the ways external tools and code connect in
  • LTI Advantage — the standard for plugging in external tools.
  • Integration Hub — deployment of those tools.
  • Valence SDKs — API libraries (Java, Python) for custom code.
› Course Design & Navigation Tools that shape the learner path. For LACE, custom HTML stays the default — reach for these only when they solve a specific need. 4 items
  • Visual Table of Contents (TOC) — a visual index of the course outline.
  • Creator+ — dynamic interactive elements (H5P and more). See the Native Tools Reference.
  • Intelligent Agents — activity-based automation and nudges.
  • Release Conditions — staged content paths that unlock as learners progress.
