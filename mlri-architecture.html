Simple flow: Brightspace to Learning Hub to Supabase

Brightspace remains the course system of record. The Learning Hub reads selected Brightspace data through secure server-side API routes, then stores the pieces we need for the custom hub experience in Supabase.

Brightspace Courses, modules, topics, enrollments, completion rules, and learner activity that Brightspace already manages.
System of record
→
Learning Hub Our custom front end and API layer. It authenticates, requests approved Brightspace data, and presents learner, manager, and admin views.
Experience layer
→
Supabase Stores the Learning Hub data we control, such as synced catalog records, search behavior, feedback, and future reporting support.

LACE Hub — one app, role-based views

Dashboards are built into LACE Hub, not as separate Brightspace, Supabase, or Monday products. A single API layer merges Brightspace + Supabase; each user sees only their role's screens.

Learner · Phase 1

My Learning

  • Enrolled courses + completion %
  • Last visited · resume link
  • Own data only (API + RLS)
Manager · Phase 3

My Team

  • Team completion rates
  • Gaps · last active
  • Scoped — not org-wide by default
