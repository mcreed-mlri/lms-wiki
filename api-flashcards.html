Meeting playbook

Use this as your spine. You already have a working spike — the goal is a production-safe Phase 1 path, not endpoint trivia.

30-second opener

LACE Learning Hub is a custom learner-facing app on Supabase/Next.js. Brightspace stays the LMS of record for enrollments, completions, and compliance. We have a local OAuth spike and can read course 6703; we need your help confirming the supported production auth model, scopes, API version, and read-only endpoint families for Phase 1 catalog sync into learning_items.

What is already proven

  • 3-legged OAuth consent and whoami work locally
  • Brightspace course/content reads work for test course 6703
  • Supabase learning_items is wired to the Learning Hub admin view
  • Next step: Service User + headless OAuth/JWKS for background sync

Suggested 45-min agenda

  1. 5 min — Context, architecture, Phase 1 scope
  2. 15 min — Auth: Service User, role, Org Unit enrollment, Client Credentials/JWKS, scopes
  3. 15 min — Data: course offering metadata, content TOC, enrollments, completion reads
  4. 7 min — Sync: API vs Data Hub/BDS, polling vs events, rate limits
  5. 3 min — Ask for recommended Phase 1 architecture and sandbox plan

Do not overshare

  • Skip deep LTI 1.3 unless they raise embedded tools
  • Defer manager dashboards and equity reporting to Phase 3
  • Keep write operations out of Phase 1 unless they recommend one
  • Vanity URL/SSL is important but secondary in an API-only session
